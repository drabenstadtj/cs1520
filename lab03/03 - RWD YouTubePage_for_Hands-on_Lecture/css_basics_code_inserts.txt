1. Responsive Weeb Design
    Responsive web design (RWD) is a web development approach that ensures 
    websites look good and function well across a wide range of devices and 
    screen sizes
    
2. Creating CSS files for Monitor, Tablet, and smartphone screen sizes
2.1. Creating a master CSS that will link your html with all the other CSS files
    In the css_basics.html file, replace the existing CSS link with the following one:

    <link rel="stylesheet" href="./static/css/screen_size_dependable.css">

2.1.1 Adding content to the screen_size_dependable.css file:

    @import url(large_screen.css) (min-width: 800px);
    @import url(med_screen.css) (min-width: 450px) and (max-width: 799px);
    @import url(small_screen.css) (max-width: 449px);

2.2. Placing the CSS files in the static/css directory 
2.2.1. Create static/css directory in your project
2.2.2. Move the existing css_basics.css to the static/css directory
2.2.3. Rename css_basics.css to large_screen.css

3. The large_screen.css is already in good shape, so we will be
    creating the med_screen.css and small_screen.css based on it.

3.1. The med_screen.css file
    Take a look at the med_screen.png

4. Create med_screen.css file in the static/css directory
4.1. Copy the contents of the large_screen.css into the med_screen.css
4.2. Decreasing the window minimum side and backgroung
    replace this

    body {
        min-width: 800px;
        background-color: aqua;
    }

    with this

    body {
        min-width: 600px;
        background-color:aquamarine;
    }

4.3. Moving the top area right div in the top bar to the next line

    replace this:

    #top_bar_right_div {
        float:right;
        margin:0px;
    }

    with this:

    #top_bar_right_div {
        clear:both;
        margin:10px 0px;
    }

4.4. Having 2 posters per column, instead of three

    #posters_ul li {
        margin: 10px;
        float:left;
        width: 45%;
        height:300px;
    }

5. Create small_screen.css file in the static/css directory
5.1. Copy the contents of the med_screen.css into the small_screen.css
5.2. Decreasing the window minimum side and backgroung

    body {
        min-width: 300px;
        background-color: cornsilk;
    }

5.3. Updating the top bar
5.3.1. Placing the mid_div in the next line (block command) by 
     removing the float:left command and adjusting the margin to
     get closer to the left window border

    #top_bar_mid_div {
        margin: 10px 0px 0px 0px;
    }

5.3.2. Same thing for the right div

    #top_bar_right_div {
        margin: 10px 0px;
    }

5.4. Removing the vertigal navigation 

    #vertical_navigation_2 {
        display: none;
    }

    and remove all the other vertical navigation rules from the CSS file

5.5. Making the right div area block command again since there is
    no vertical navigation bar anymore

    replace this:

    #right_area_div {
        float:left;
        width:80%;
    }

    with this:

    #right_ area_div {
        padding: 0px;
        margin: 0px;
    }

5.6. Updating the horizontal navigation bar

    #horizontal_navigation {
        clear:both;
        list-style-type: none;
        padding: 0px;
        margin: 0px;
    }

5.7. Updating the poster <ul>
5.7.1. Bringing the posters closed to the left border of the page

    #posters_ul {
        clear: both;
        list-style-type: none;
        padding: 0px;  // <==
        margin: 0px;   // <==
    }

5.7.2. Setting the li to take all the space of a window
    remove float:left and width:xyz%. <li> will be a block command
    for smarthphone

    #posters_ul li {
